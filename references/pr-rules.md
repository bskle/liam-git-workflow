# PR Rules

## Minimum Content

1. 变更目的与背景
2. 主要改动点
3. 风险与影响范围
4. 测试说明
5. 回滚方案

## Merge Expectations

- at least one reviewer approval when review is available
- CI should be green
- unresolved review comments block merge
- prefer squash merge

## Finish Checklist

- current branch targets the correct merge branch
- latest base branch has been synced
- no sensitive data or temporary files
- commit history is understandable
- release or hotfix flow is followed when needed
