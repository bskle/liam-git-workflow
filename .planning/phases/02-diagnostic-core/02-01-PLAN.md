### 4. 核心辅助函数 Invoke-DiagnosticCommand

实现 RESEARCH.md §Architecture Patterns Pattern 2 的**完整版本**：

```powershell
function Invoke-DiagnosticCommand {
    param(
        [string]$CommandName,
        [scriptblock]$ScriptBlock
    )
    # 1. 保存当前 $ErrorActionPreference
    # 2. 设置 $ErrorActionPreference = 'Continue'（关键：防止 2>&1 bug）
    # 3. 执行 ScriptBlock：& $ScriptBlock 2>&1 | Out-String
    # 4. 捕获 $LASTEXITCODE（在 ScriptBlock 执行后立即读取）
    # 5. try/catch 包裹执行——如果 ScriptBlock 抛出异常，
    #    捕获异常消息为 stderr，exitCode 设为 -1
    # 6. 恢复 $ErrorActionPreference
    # 7. 返回 @{ Output = <string>; ExitCode = <int>; Error = <string|null> }

    # 关键约束：
    # - 不抛出终止错误（不中断调用者）
    # - 不清空 $LASTEXITCODE（在执行后立即读取）
    # - 不对 Output 的内容做假设（可能为空、多行、包含特殊字符）
}
```

### 6. 诊断层 1：Test-LocalRepositoryState

实现 RESEARCH.md §Architecture Patterns Pattern 1（乐观收集）的第 1 层：

```powershell
function Test-LocalRepositoryState {
    param(
        [string]$RepoPath
    )
    # 创建本地 $evidence 和 $actions 数组
    # 返回 @{ Evidence = @(); Actions = @(); Findings = @{} }

    # 按顺序执行以下检查（每个检查调用 Invoke-GitCommand 或检查文件系统）：
    # a) git branch --show-current → 判断是否 detached HEAD（空输出）
    #    - 正常：返回分支名 → 记录 evidence
    #    - detached HEAD：空输出 → 记录 evidence + findings['detached_head'] = $true
    #
    # b) 检查 .git/MERGE_HEAD 文件是否存在（Join-Path $RepoPath '.git' 'MERGE_HEAD'）
    #    → 存在 = merge 进行中，记录 evidence + findings['merge_in_progress'] = $true
    #
    # c) 检查 .git/rebase-merge 和 .git/rebase-apply 目录是否存在
    #    → 任一存在 = rebase 进行中，记录 evidence + findings['rebase_in_progress'] = $true
    #
    # d) git status --porcelain → 检查输出中是否包含 'UU '（未解决的合并冲突）
    #    → 包含 UU → 记录 evidence + findings['has_conflicts'] = $true
    #    → 还检查是否有其他未提交变更（M, A, D, R 状态），记录 findings['has_uncommitted_changes']
    #
    # e) git branch -vv → 检查当前分支是否有 upstream（查找 '* branch_name ... [remote/branch]' 模式）
    #    → 没有 [...] = 未设置 upstream → 记录 evidence + findings['no_upstream'] = $true
    #    → 有 [...] 但包含 ': gone]' = 远程分支已删除 → 记录 evidence + findings['upstream_gone'] = $true
    #
    # 每个检查的结果同时添加到 $evidence 数组和 $actionsTaken 数组
    # evidence 格式：描述型字符串（中文）
    #   示例："当前分支: feature/test（正常）"
    #   示例："检测到 merge 进行中（.git/MERGE_HEAD 存在）"
    # actions 格式：命令 + 结果
    #   示例："git branch --show-current → feature/test"
    #   示例："Test-Path .git/MERGE_HEAD → True（merge 进行中）"
}
```

**验证点：**
- 所有 git 命令通过 `Invoke-GitCommand` 执行（不是直接调用 git）
- `$evidence` 和 `$actions` 使用中文描述
- 文件系统检查使用 `Join-Path $RepoPath '.git' '<file>'` 构建路径

```powershell
function Test-RemoteTarget {
    param(
        [string]$RepoPath,
        [string]$RemoteName
    )
    # 返回 @{ Evidence; Actions; Findings }

    # 按顺序执行：
    # a) git remote -v → 解析 fetch 和 push URL
    #    - 输出为空 → findings['no_remotes'] = $true，
    #      记录 evidence："未配置任何远程仓库"
    #    - 有输出 → 提取指定 $RemoteName 的 fetch URL
    #      → findings['remote_url'] = <URL 字符串>，
    #        记录 evidence："远程仓库 origin URL: https://github.com/user/repo.git"
    #
    # b) 验证 remote URL 格式（在 findings['remote_url'] 存在时）：
    #    检查是否匹配 https://<host>/<owner>/<repo>.git 或 git@<host>:<owner>/<repo>.git
    #    - 格式正确 → 记录 ParsedHost, ParsedOwner, ParsedRepo
    #    - 格式错误 → findings['url_malformed'] = $true，
    #      记录 evidence："远程 URL 格式异常: <url>"
    #    - 协议不支持（非 https/ssh）→ 记录 evidence
    #
    # c) git branch -vv → 检查当前分支的 upstream 映射
    #    - 查找 '* <branch> ... [<remote>/<upstream>]' 格式
    #    - upstream 与 $RemoteName 不匹配 → 记录 evidence
    #    - 包含 ': gone]' → findings['upstream_gone'] = $true
    #
    # d) git ls-remote --heads <RemoteName> → 轻量远程连通性测试
    #    - 成功（exit 0）→ 解析分支引用列表
    #      记录 evidence："git ls-remote origin --heads 成功，发现 X 个分支"
    #    - 失败 → 记录 exit code + stderr 到 evidence
    #      不中断——这是诊断数据，后续层会进一步分析
}
```

**关键约束：** `git ls-remote` 失败是诊断数据，不是脚本终止条件。其输出需要继续传递给后续信号匹配。

```powershell
function Test-Authentication {
    param(
        [string]$RepoPath,
        [string]$RemoteName,
        [string]$RemoteUrl
    )
    # 返回 @{ Evidence; Actions; Findings }

    # 按顺序执行：
    # a) 确定传输类型：
    #    检查 $RemoteUrl 前缀：
    #    - 以 'https://' 开头 → findings['transport'] = 'HTTPS'
    #    - 以 'git@' 开头 → findings['transport'] = 'SSH'
    #    - 其他 → findings['transport'] = 'unknown'
    #    记录 evidence："传输类型: HTTPS"
    #
    # b) 检查凭据助手配置：
    #    git config --get credential.helper
    #    - 有输出 → 记录 evidence："Git 凭据助手: manager-core"
    #    - 空 → findings['no_credential_helper'] = $true，
    #      记录 evidence："未配置 Git 凭据助手"
    #
    # c) HTTPS 传输类型时：
    #    git config --get --show-origin credential.helper
    #    → 显示配置来源（全局/本地），记录 evidence
    #
    # d) SSH 传输类型时：
    #    - 检查 TCP 22 可达性：Test-NetConnection <host> -Port 22 -WarningAction SilentlyContinue
    #      记录 TcpTestSucceeded 结果到 evidence
    #    - 如果 TCP 22 可达 → 执行 ssh -o ConnectTimeout=5 -T <git_user>@<host>
    #      使用 Invoke-DiagnosticCommand 执行，超时 5 秒（见 RESEARCH.md Pitfall 5）
    #      → 成功认证消息（"successfully authenticated"）→
    #         记录 evidence："SSH 认证成功 (ssh -T git@github.com)"
    #      → Permission denied → findings['ssh_auth_failed'] = $true
    #         记录 evidence："SSH 认证失败: Permission denied"
    #    - 如果 TCP 22 不可达 → findings['ssh_port_blocked'] = $true
    #      记录 evidence："SSH 端口 22 不可达——可能被防火墙阻止"
    #      **不执行** ssh -T（避免挂起——见 RESEARCH.md Pitfall 5）
    #
    # e) 检查失败命令 stderr 中的认证错误签名：
    #    如果 $script:Stderr 变量非空（脚本级变量）：
    #    - 匹配 'fatal: Authentication failed' → findings['auth_error_401'] = $true
    #      记录 evidence："stderr 包含认证失败签名: Authentication failed"
```

```powershell
$allActions += $layer4.Actions

# - 系统证书问题（超出仓库 CLI 范围）

$humanDetail = $null

#   → $needsHuman = $true
#   → 使用 references/remote-diagnostics.md §升级与人工交互边界 中对应场景的用户提示模板
# - Category 为 "TLS/SSL 错误" + findings 中无代理配置 → 可能需要检查系统证书
#   → $needsHuman = $true
#   → 提示用户使用 certmgr.msc 检查证书存储
# - 其他 Category（DNS 失败、连接超时、认证失败、代理错误、本地状态阻塞）→ CLI 可诊断
#   → $needsHuman = $false

        $needsHuman = $true
        $humanDetail = "无法从命令行确认仓库策略的具体配置。请在浏览器中检查仓库设置：1) 访问仓库的 Settings → Branches，查看分支保护规则；2) 确认是否启用了 'Require a pull request before merging'。回报：分支保护规则的具体配置（是否要求 PR、是否要求审查、是否要求状态检查）。"

    '仓库不存在' {
        $needsHuman = $true
        $humanDetail = "无法从命令行确认仓库的当前状态。请在浏览器中：1) 访问仓库 URL，确认仓库是否存在；2) 检查仓库是否已被删除、重命名或归档；3) 确认您的账号是否有该仓库的访问权限。回报：仓库是否存在、仓库的可见性（public/private）、您的访问权限级别。"
    }

        $humanDetail = "无法从命令行确认您的账号授权范围。请在浏览器中：1) 访问 GitHub Settings → Personal Access Tokens，检查当前 Token 的权限范围（至少需要 'repo' 权限）；2) 如果是组织仓库，访问组织 SSO 授权页面确认 SSO 认证状态。回报：Token 的权限范围，以及 SSO 认证状态（已认证/未认证）。"
    }

        $needsHuman = $true
        $humanDetail = "无法从命令行确认系统证书存储状态。此问题可能由企业自签名证书或代理 SSL 拦截引起。请在 Windows 上：1) 打开 certmgr.msc；2) 浏览到 '受信任的根证书颁发机构' → '证书'；3) 查找是否有企业 CA 证书存在。回报：是否存在企业 CA 证书，以及证书是否未过期。"
    }
    default {
        $needsHuman = $false
        $humanDetail = $null
    }
}

    '本地状态阻塞' {
        if ($allFindings.ContainsKey('merge_in_progress')) { $nextAction = '执行 git merge --abort 中止当前合并' }
        elseif ($allFindings.ContainsKey('rebase_in_progress')) { $nextAction = '执行 git rebase --abort 中止当前变基' }
        elseif ($allFindings.ContainsKey('detached_head')) { $nextAction = '执行 git checkout <branch-name> 切换到已有分支' }
        elseif ($allFindings.ContainsKey('no_upstream')) { $nextAction = '执行 git branch --set-upstream-to=origin/<branch> <branch> 设置 upstream' }
        else { $nextAction = '解决本地仓库状态问题后重试远程操作' }
    }
    'DNS 解析失败' { $nextAction = '检查 DNS 配置：1) 尝试 nslookup github.com；2) 如使用企业网络，联系 IT 确认 DNS 过滤策略；3) 可尝试修改 DNS 为 8.8.8.8 或 1.1.1.1' }
    '连接超时/拒绝' { $nextAction = '检查网络连通性：1) 确认是否有 VPN 或代理运行；2) 如使用 SSH 且端口 22 被阻，执行 git remote set-url origin https://... 切换为 HTTPS' }
    '认证失败' { $nextAction = '刷新 Git 凭据：1) 执行 git credential reject 清除旧凭据；2) 重新执行远程操作触发认证弹窗；3) 如使用 SSH，验证密钥已添加到 GitHub' }
    '授权不足' { $nextAction = '检查访问权限——需要人工在浏览器中验证 Token 权限和组织 SSO 状态（见 human_action_detail）' }
    '代理错误' { $nextAction = '修复代理配置：1) 检查 git config --global http.proxy 是否正确；2) 如不需要代理，执行 git config --global --unset http.proxy 移除代理；3) 检查 Windows 系统代理设置' }
    'TLS/SSL 错误' { $nextAction = '检查证书配置——需要人工在 Windows certmgr.msc 中验证证书链（见 human_action_detail）。临时绕过（风险操作）：git config --global http.sslVerify false' }
    '仓库策略拒绝' { $nextAction = '仓库策略限制——需要通过 PR 合并或请求管理员权限。具体操作见 human_action_detail。' }
    '仓库不存在' { $nextAction = '验证仓库存在性——需要人工在浏览器中确认仓库状态（见 human_action_detail）。如仓库已迁移，更新远程 URL：git remote set-url origin <new-url>' }
    '远程目标错误' {
        if ($allFindings.ContainsKey('no_remotes')) { $nextAction = '添加远程仓库：git remote add origin <repository-url>' }
        elseif ($allFindings.ContainsKey('url_malformed')) { $nextAction = '修正远程 URL：git remote set-url origin <correct-url>' }
        elseif ($allFindings.ContainsKey('upstream_gone')) { $nextAction = '远程分支已删除：执行 git fetch --prune 清理本地引用' }
        else { $nextAction = '检查并修正远程仓库配置后重试' }
    }
    default { $nextAction = '诊断未匹配到已知模式——请人工分析以上 evidence 输出，并将发现反馈给诊断系统' }
}

# Step 6: 构建并输出最终 JSON
$output = Build-StructuredOutput `
    -ProblemCategory $rootCause.Category `
    -Evidence $allEvidence `
    -LikelyCause $rootCause.Cause `
    -Confidence $rootCause.Confidence `
    -ActionsTaken $allActions `
    -RecommendedNextAction $nextAction `
    -HumanInteractionRequired $needsHuman `
    -HumanActionDetail $humanDetail

Write-Output $output
exit 0
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User Input (parameters) → Script Internals | -RepoPath 接受任意路径字符串；-Stderr 接受任意文本。必须验证和消毒。 |
| Script → stdout (JSON output) | JSON 输出供主 Agent 消费——必须不包含凭据或敏感数据。 |
| Script → stderr (diagnostic logs) | 诊断日志可能包含文件路径信息，但不包含凭据内容。 |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Information Disclosure | evidence array / JSON stdout | mitigate | 过滤 evidence 中潜在的凭据泄露模式（`ghp_` token 前缀、`github_pat_` 前缀、`Authorization: Basic` 头）。脚本仅记录凭据助手名称（如 `manager-core`），不读取凭据内容。 |
