<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| plugin.json → Claude Code validator | plugin.json 内容被 Claude Code 严格校验；无效字段导致安装拒绝 |
| hooks.json → Claude Code hook 执行引擎 | hooks.json 中的 command 字段在用户机器上以 PowerShell 执行；命令注入风险 |
| 仓库源码 → 用户文件系统 | 仓库被 clone 到本地后作为可信插件源；恶意修改的 plugin.json/hooks.json 可执行任意命令 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | plugin.json 完整性 | accept | plugin.json 是静态 JSON 文件，随 git 仓库分发。仓库本身是 git clone 的本地副本，篡改需要本地文件系统访问权限（不在威胁模型内）。通过 git 标签/commit hash 可验证来源。 |
| T-04-02 | Elevation of Privilege | hooks.json SessionStart command | mitigate | hooks.json 中的 PowerShell command 以用户权限执行。命令使用硬编码字符串输出 JSON，不接收任何外部输入，不接受任何参数，不调用外部脚本。无需缓解措施额外加固（ASVS V5.2.4: 无用户输入的命令执行）。 |
| T-04-03 | Information Disclosure | plugin.json metadata | accept | plugin.json 仅包含非敏感元数据（name, version, description, author, keywords）。无密钥、令牌、环境变量或 PII。 |
| T-04-04 | Spoofing | plugin.json name 字段 | accept | name 字段 "liam-git-workflow" 用于本地插件识别，非安全关键标识。Claude Code 插件系统不执行基于 name 的权限检查。 |
| T-04-05 | Denial of Service | hooks.json matcher 模式 | accept | matcher "startup|clear|compact" 触发频率低（仅在会话事件时触发）。command 是同步的（async: false），但执行的是轻量级 PowerShell echo（毫秒级），不阻塞会话启动。 |
| T-04-06 | Elevation of Privilege | hooks.json command 注入向量 | mitigate | command 字符串为静态硬编码值（"powershell -NoProfile -ExecutionPolicy Bypass -Command \"Write-Output '...'\""），不从任何外部输入构建，不接受参数插值。ASVS V5.2.4 满足。即使 plugin.json 被恶意修改引入注入 payload，攻击者已需要本地文件系统写入权限（超出威胁模型）。 |
</threat_model>
